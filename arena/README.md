## DSD Arena

Интерактивная страница-«арена» поверх видео: выбираешь персонажа — подсветка по маске показывает, кого выбрал; кликом запускается его короткий «трюк‑приветствие». Поверх сцены есть мини‑терминал, который пишет события выбора.

### Задумка

- **Цель**: сделать ощущение «выбора бойца» как в аркадных/arena‑шутерах — но на базе реального видео.
- **UX**:
  - **Hover/Move** (мышь): подсветка маски + имя над персонажем.
  - **Click/Tap**: запуск трюка выбранного персонажа.
  - Мини‑терминал сообщает: «Вы выбрали персонажа…», «Запуск трюка…».

### Технические характеристики ассетов

- **Видео**: `arena_full.mp4` (1280×720, ~62.5s)
- **Faststart-вариант**: `arena_full_faststart.mp4` (moov atom в начале для веб‑загрузки)
- **Маски** (1280×720): `arena/mask/*.png`
  - Чёрное = персонаж, белое = фон.
  - Порядок слоёв (сзади → вперёд): Ivan → Slavok → Hep → Yurtso.

### Персонажи и тайминги (исходное видео)

- **Hep**: `0:04–0:10`
- **N.J.**: `0:16–0:22` (удлинено на 1s)
- **Custo**: `0:35–0:41`
- **$WK**: `0:53–0:58`

### Почему используются клипы, а не seek по `currentTime`

В некоторых окружениях/серверной отдаче MP4 браузер может **не уметь прыгать по времени** (seek «залипает» на `0.00`). Чтобы поведение было 100% стабильным, по клику мы **переключаем `video.src` на отдельный клип** нужного трюка:

- `arena/clips/hep.mp4`
- `arena/clips/yurtso.mp4`
- `arena/clips/ivan.mp4`
- `arena/clips/slavok.mp4`

Каждый клип играет с `0:00` (уже являясь нужным отрезком из исходника).

### Как реализовано

- **Hit-test по маске (pixel‑perfect)**:
  - Маска загружается в offscreen canvas → `ImageData`.
  - На `mousemove` координата курсора маппится в 1280×720.
  - Проверяем пиксель маски (тёмный/не тёмный) в порядке слоёв «спереди‑назад».
- **Подсветка**:
  - Для каждого персонажа собирается кешированный highlight canvas (`hl`), чтобы не лагало.
  - Рисуется поверх видео через overlay canvas.
- **Частицы**:
  - Для каждой маски заранее строится редкий список точек внутри маски (grid step ~6px).
  - На hover спавним немного частиц и анимируем их через `requestAnimationFrame`.
- **Терминал**:
  - Блок `.arena-terminal` в левом верхнем углу.
  - Лог хранит последние ~14 строк.
- **Контролы**:
  - `Original/Clips`: оригинальное видео vs режим клипов персонажей.
  - `🔊/🔇`: mute.
  - `⏯`: play/pause.

### Структура проекта

- `arena/index.html` — разметка и подключения.
- `arena/css/arena.css` — стили.
- `arena/js/arena.js` — логика выбора/видео/FX/терминала.
- `arena/mask/*.png` — маски персонажей.
- `arena/clips/*.mp4` — нарезанные трюки.

### Debug в консоли

После загрузки страницы в консоли выводится приветствие и подсказка по debug-режиму.

- Включить:
  - `debugArena = true`
  - или `debugArena = True` (поддерживается специально для удобства)
- Выключить:
  - `debugArena = false`

Когда debug включён, выводятся служебные логи:
- клик по персонажу и его диапазон;
- выбор/подмена `video.src`;
- загрузка клипа (`loadeddata`) и `duration/currentTime`;
- завершение клипа;
- переключение режимов `Original/Clips`;
- автоскрытие подсветки;
- инициализация масок/сцены.

### Обновление клипов (через ffmpeg)

Примеры команд (Windows/Powershell):

```bash
# Faststart (перенести moov atom в начало)
ffmpeg -y -i "arena/arena_full.mp4" -c copy -movflags +faststart "arena/arena_full_faststart.mp4"

# Нарезка клипа (пример N.J. 0:16–0:22)
ffmpeg -y -ss 00:00:16 -to 00:00:22 -i "arena/arena_full_faststart.mp4" -c:v libx264 -preset veryfast -crf 20 -c:a aac -b:a 160k "arena/clips/yurtso.mp4"
```
